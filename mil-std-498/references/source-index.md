# Source index

This skill was designed from public MIL-STD-498 descriptions and the VCTLabs GitHub repository structure. It does not bundle the upstream PDF files; it provides workflow instructions, DID cataloging, skeleton generation, and review checklists.

## Upstream repository

- Repository: https://github.com/VCTLabs/MIL-STD-498
- README describes MIL-STD-498 systems/software engineering document templates in PDF and HTML, plus guidebook material.
- Repository PDF directory includes the MIL-STD-498 standard PDF, GuideBook PDFs, and DID PDFs such as SRS, SDD, SDP, STP, STD, STR, SUM, SVD, SPS, IRS, IDD, DBDD, SSS, SSDD, OCD, SIP, STRP, COM, CPM, FSM, SCOM, SIOM, and an ICD-related PDF.

## Important interpretation notes

- MIL-STD-498 is a legacy/canceled standard, but it remains useful as a rigorous documentation framework for software lifecycle work.
- The skill does not reproduce full DID text. For formal compliance, obtain the exact controlling DID, CDRL, SOW, and customer tailoring.
- If the user provides upstream PDFs or contract templates, use those over this skill's generic structures.
